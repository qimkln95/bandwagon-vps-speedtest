## 搬瓦工VPS速度测试到底测什么

很多人把"延迟"和"速度"混着说，但这是两件不同的事。

**延迟（Ping / RTT）**，单位毫秒（ms），代表你发一个数据包到服务器再回来需要多久。延迟跟服务器的物理位置和线路质量直接相关，不是你能通过任何"优化"改掉的。

**丢包率**，代表传输中有多少数据没到达对端。丢包比延迟高更影响体验——延迟 200ms 你可能不觉得，丢包 10% 就会感觉卡顿、重传。

**下载/上传速度**，也就是通常说的"测速"，由带宽上限和当前网络质量共同决定。搬瓦工不同机房带宽不同，CN2 GIA-E 系列的带宽上限是 2.5Gbps，香港 CN2 GIA 是 1Gbps，普通 KVM 套餐是 1Gbps。

还有一个进阶指标是**路由路径**。走 CN2 GIA（AS4809）还是走普通 163 线路（AS4134），影响巨大。测速数字看起来差不多，晚高峰丢包可能差很远。

## 各机房速度怎么样：延迟参考范围

不同机房、不同运营商，延迟差距很大。以下是实际使用中大致的参考范围：

**中国香港 CN2 GIA（HKHK_8）**：物理距离近，广东地区延迟可以低到 10ms 上下，北方地区在 30–50ms 之间。三网全程 CN2 GIA，丢包控制在同价位里是最好的。缺点就是贵，而且这个机房不支持切换，买了就绑死在香港。

**日本大阪软银（JPOS_1）**：联通用户直连软银，延迟非常稳，在 80ms 左右。电信用户走 CN2 GIA 转接，也不算差。移动用户一般建议看 DC6 / DC9。

**洛杉矶 DC6 CN2 GIA-E（USCA_6）**：电信、联通双程 CN2 GIA，移动走 CMIN2 回程。实测晚高峰基本不掉速，这也是 CN2 GIA-E 套餐的默认机房之一。从国内 ping 过去一般在 160–180ms，这个延迟对洛杉矶来说算正常，丢包表现比 CN2 GT 好得多。

**洛杉矶 DC9 CN2 GIA（USCA_9）**：和 DC6 类似，都是 CN2 GIA 线路，差别在于 DC9 带宽是 1Gbps，DC6 是 2.5Gbps。电信、联通用双程 CN2 GIA，移动走 CMIN2。性能稍逊但够用，也是 CN2 GIA-E 套餐可以切换的机房之一。

**KVM 套餐（普通线路）**：默认在洛杉矶 DC2 或弗里蒙特（FMT），走普通国际线路，延迟和丢包在晚高峰有时候波动比较明显。适合预算紧、对速度要求不高的用户。

## 买后怎么测：3 种方法

### 方法一：用搬瓦工官方演示站点（SpeedTest）

部分机房有搬瓦工自建的 SpeedTest 页面，可以直接在浏览器里测速：

- DC9 演示站：`dc9.bwg.net`
- DC2 演示站：`dc2.bwg.net`
- 日本大阪演示站：`jpos.bwg.net`

打开对应站点，点击"开始测试"，能看到下载速度、上传速度、延迟和延迟波动。这是最直观的方式。

### 方法二：VPS 内部跑测速脚本

登录到你的搬瓦工 VPS 后，可以用 bench.sh 脚本做一次完整测试：
bash
curl -Lso- bench.sh | bash

这会输出硬件信息、各大测速节点的下行速度，让你看到服务器本身的性能和多线路下行带宽。

### 方法三：用 NextTrace 测回程路由

买完之后最值得跑一次的是路由跟踪，确认你的流量是不是真的走 CN2 GIA。
bash
# 安装 NextTrace
curl nxtrace.org/nt | bash

# 测试到电信骨干的路径
nexttrace 219.141.136.10

# 测试到联通的路径
nexttrace 202.106.0.20

如果测试结果里中间节点显示 `59.43.x.x`（AS4809），那就是 CN2 GIA。显示 `202.97.x.x`（AS4134）就是普通 163 线路，差距不小。
